# Retail Sales Analytics & Demand Forecasting

## Project Overview

This project focuses on exploratory data analysis and time series forecasting of retail transaction data.

The goal is to analyze sales dynamics, identify patterns in customer behavior, and build a forecasting model for revenue prediction.

## Key Tasks

- Data cleaning and preprocessing
- Handling missing values and outliers
- Exploratory Data Analysis (EDA)
- Feature engineering (time-based features, purchase segmentation)
- Statistical hypothesis testing
- Customer and sales segmentation analysis
- Time series analysis (stationarity testing, ACF/PACF)
- Demand forecasting using ARIMA model

## Results

- Identified sales trends over time
- Analyzed differences between stores, time periods, and purchase segments
- Performed RFM-style customer segmentation
- Built a baseline ARIMA forecasting model for revenue prediction

## Tools

- Python
- pandas
- matplotlib
- scipy
- statsmodels
- plotly

## Dataset

The project uses a sample retail dataset from IBM Cognos Analytics, representing transactional data from a fictional coffee shop chain in New York.

It simulates real-world retail operations including customer purchases, store performance, and product-level transactions.

The dataset contains multiple related tables:
- Sales receipts
- Products
- Customers
- Staff
- Sales outlets
- Inventory and dates

This dataset is intended for analytical and educational purposes.

Source: IBM Sample Data (Coffee Chain Dataset)
